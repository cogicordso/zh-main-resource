# zh-main-resource

## 项目简介

本仓库用于归档和发布多个独立的 HTML 页面。这些页面涵盖了不同主题和用途，旨在提供简洁、可直接使用的静态资源。仓库本身不针对任何特定域名或网站，仅作为页面分类存放和版本管理的场所。

## 目录结构

```
zh-main-resource/
├── pages/          # 存放各独立 HTML 页面文件
├── assets/         # 页面所需的共用资源（如 CSS、JS、图片等）
└── README.md       # 本说明文件
```

- `pages/`：每个 HTML 文件为一个独立页面，命名清晰，便于识别内容。
- `assets/`：页面引用的外部资源，按需组织，不包含敏感或隐私信息。

## 页面归档说明

- 所有页面均为静态 HTML，无需后端支持，可直接在浏览器中打开。
- 页面内容以信息展示、工具或文档为主，不包含交互式服务或动态数据。
- 归档遵循“一页一文件”原则，避免页面间依赖，便于单独使用或分享。
- 页面更新时，旧版本将保留在历史提交中，当前目录始终为最新版本。

## 维护说明

- 本仓库由维护者不定期更新，新增或修改页面均会通过 Pull Request 或直接提交。
- 欢迎提交 Issue 反馈问题或建议，但请避免讨论具体域名或商业用途。
- 不接受包含广告、跟踪脚本或恶意代码的页面提交。
- 若需引用或分发本仓库中的页面，请遵守开源许可证（见 LICENSE 文件）。

## 使用方式

1. 克隆或下载本仓库到本地。
2. 在 `pages/` 目录下找到目标 HTML 文件。
3. 用浏览器直接打开该文件，即可查看页面内容。
4. 也可将页面部署至任意静态托管服务（如 GitHub Pages、Netlify 等）。
